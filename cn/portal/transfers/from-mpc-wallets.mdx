本文介绍 MPC 钱包（机构钱包）的提币流程，并指导您如何在 Cobo Portal 中发起提币。

## 前提条件

- 建议您设置[链下交易风控规则](/cn/portal/risk-controls/off-chain-tx-set-transfer)以确保交易安全。
- 您已向 MPC 钱包（机构钱包）[充币](/cn/portal/mpc-wallets/ocw/transfers)。

## 提币流程

下图展示了通过 Cobo Portal 和 WaaS API 从机构钱包提币的流程。

<img src="/cn/images/transfers/mpc-withdrawal-process.svg" className="diagram" alt="从机构钱包提币的流程"/>

1. **发起提币请求**：用户通过 Cobo Portal 或 [WaaS API](https://www.cobo.com/developers/v2/guides/mpc-wallets/get-started-ocw#withdraw-tokens) 发起提币请求。
2. **二次确认**：
   - 通过 Cobo Portal 发起的提币：需要在 [Cobo Guard](/cn/guard/introduction) 上确认提币请求。
   - 通过 WaaS API 发起的提币：您的 [Callback Endpoint](https://www.cobo.com/developers/v2/guides/webhooks-callbacks/introduction) 需要返回 "ok" 以确认提币请求。
3. **[风控](/cn/portal/risk-controls/tx-policies-overview)**：如果已配置，Cobo Portal 中的交易风控将评估提币请求。当需要人工审核时，审批员需要在 Cobo Guard 上审批该交易。
4. **签名**：Cobo 根据在线状态等条件选择一个可用的[签名人](/cn/portal/mpc-wallets/ocw/holder-group-main-group#create-a-main-group)来签署交易。请确保至少有一个签名人在线。您可以通过 Cobo Portal 查看签名人是否在线。详情请参见本文**查看签名人状态**一节。
5. **广播**：Cobo 创建交易并将其广播到网络。


## 发起提币

要从机构钱包转账，请登录 [Cobo Portal](https://portal.cobo.com/login)，然后点击左侧主菜单中的 <img src="/cn/images/transfers/transfer-button.svg" className="icon"></img>。

<img src="/cn/images/transfers/transfer-button-location.png" className="screenshot_modal"/>

<Note>在 Cobo Portal 上，单次提币只能有一个接收地址。要一次性提币到多个接收地址，请使用 [Transfer token API 操作](https://www.cobo.com/developers/v2/api-references/transactions/transfer-token)。</Note>

然后填写所需的信息，具体如下：

1. 选择转账的代币类型。可选代币类型仅限于团队账户的订阅包中所含的代币。要启用新的代币类型（额外收费），请参阅[从外部钱包充币](https://manuals.cobo.com/cn/portal/mpc-wallets/ocw/transfers#)中的步骤 1 到 6。

2. 在“出金地址”选项的下拉菜单中选择 **MPC 钱包** -> **机构钱包**。

3. 要找到您的钱包，请从下拉菜单中选择，也可以直接搜索地址、Vault 名称、钱包名称或标签。结果中将列出通过 Cobo Portal 和 WaaS 2.0 API 创建的 MPC 钱包（机构钱包）。

4. 从地址簿或您在 Cobo Portal 上的钱包中选择**接收地址**，也可以手动输入地址。
    - **地址簿**<img src="/en/images/common/icon_address_book.svg" className="icon" alt="Address Book icon" />：仅列出与所选代币的区块链网络相关联的地址。
    - **我的钱包**：选择钱包类别，然后点击您希望转入的钱包。
5. 输入转账金额。注意，对于 DOT、XRP 或 KSM 代币，所选的 MPC 钱包（机构钱包）中持有的余额必须满足最低要求。DOT 代币的最低余额为 1 DOT，XRP 为 10 XRP，KSM 为 0.000333333 KSM。
6. 查看交易费用。根据交易速度，网络费用分为三个级别：慢、标准和快。
7. 要调整该费用，请点击下图圈出的按钮。所有字段都是必填项，并将根据您选择的交易速度自动填充。您可以根据自己的具体要求编辑各个参数。
    <img src="/cn/images/transfers/customize-fees.png" className="screenshot_modal"/>
    - EVM 链：对于 EVM 链，EIP1559 和 Legacy 方式的交易费用有所不同。
        - EIP 1559
            - 在 EIP1559 方式下，用户可以定制 3 个参数：Max Priority Fee (gwei)、Max Fee 和 Gas Limit。
            - 预估费用的计算方式是 Max Fee 乘以 Gas Limit：（Max Fee x Gas Limit）。
            - 对于 Mantle 和 Optimism，显示的费用仅涵盖 Layer 2。提交交易后，Layer 1 费用将自动从您的出金地址扣除。请确保您的地址有足够的 Gas 代币以支付所有费用。
            - Chiliz 不收取费用，但 Chiliz Chain 2.0 收费。

            <a id="eip1559"></a>	
            | 参数        | 定义                                                                                                                                                                                                                             |
            | --------         | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------  |
            | Max Priority Fee | Max Priority Fee 通常也被称为“矿工小费”，支付给矿工，激励他们优先处理您的交易。Max Priority Fee 是您愿意为交易支付的最高矿工小费。
            | Max Fee          | Max Fee 是您愿意为交易支付的 Gas 总价。                                                                                                                                                |
            | Gas Limit        | Gas Limit 是您愿意为执行交易或运行以太坊虚拟机 (EVM) 而支付的最高 Gas Unit 数量。不同的运行方式需要消耗不同数量的 Gas Unit。              |

        - Legacy
            - 在 Legacy 方式下，用户可以定义 2 个参数：Gas Price (gwei) 和 Gas Limit。
            - 预估费用的计算方式是 Gas Price 乘以 Gas Limit：（Gas Price x Gas Limit）。

            | 参数 | 定义                                                                                                                                                  |
            | --------  | ----------------------------------------------------------------------------------------------------------------------------------------------------------  |
            | Gas Price | Gas Price 是必须支付给验证者以便处理交易的 ETH（单位较小，称为 gwei）数量。                   |
            | Gas Limit | Gas Limit 是您愿意为执行交易或运行以太坊虚拟机 (EVM) 而支付的最多 Gas Unit 数量。 |                                                                                              |
    - 基于 UTXO 的链
        - 用户可以自定义费率 (sat/vByte)。
        - 预估费用的计算方式是费率乘以交易数据量：（费率 x 交易数据量）。

            | 参数 | 定义                                                                                                                               |
            | --------  | -----------------------------------------------------------------------------------------------------------------------------------------|
            | 费率  | 费率的计算方式是您的交易在区块链上消耗的每单位数据对应多少 satoshis（缩写为 sats/vByte）。|
8. <MySnippet />
9. 点击**提交**。
10. 在 Cobo Guard 中批准该请求。如果该地址为新地址，不在您的地址簿或 Cobo Portal 上创建的钱包中，系统将提示您将该地址添加到地址簿。

## 查看签名人状态

要成功签名交易，您需要确保至少有一个签名人在线。交易将保持 Pending 状态，直到收到必要的签名。

您可以通过 Cobo Portal 查看签名人是否在线：
1. 登录 [Cobo Portal](https://portal.cobo.com/login)。
2. 点击 <img src="/cn/images/transfers/wallets-icon.svg" className="icon"></img> > **MPC 钱包**。
3. 选择您的 Vault，然后点击右上角的 <img src="/cn/images/transfers/manage-key-share-group.svg" className="icon"></img> 图标。
4. 在**私钥分片管理**页面，找到您的私钥分片持有者组，并查看每个私钥分片持有者旁边的状态指示器：
   - 🟢 绿点：签名人在线并准备签署交易
   - 无绿点：签名人离线，无法签署交易。
