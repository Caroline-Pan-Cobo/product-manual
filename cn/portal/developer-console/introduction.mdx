Cobo Portal 的 开发者控制台是您的团队在 API 与集成相关事务的控制中心。无论您是在构建 Cobo Portal Apps 还是其他 WaaS 2.0 App，开发者控制台都能帮助您从 API Key 创建到 Webhook 稳定性监控，全流程保持运行顺畅。

控制台分为两个部分，每个部分都提供了适用于不同开发场景的功能：

## WaaS 2.0

该标签页用于监控和管理与 WaaS 2.0 服务相关的 API 操作，为您提供团队后端 API 流量和集成的全面可视性。

在 WaaS 2.0 标签页中，您可以：

- **概览面板**：以可视化方式展示实时 API 使用情况、调用最多的接口以及请求分布，帮助您了解流量模式或识别异常。
- **API Key**：创建、编辑、撤销和监控团队中用于访问 WaaS 2.0 的 API Key。您还可以注册 Callback Endpoint，并将其与特定 API Key关联。
- **Webhook**：注册、编辑或撤销 Webhook Endpoint，并查看事件历史。
- **API 日志**：可搜索的日志查看器，记录所有 API 请求与响应，适用于调试和审计。
- **Webhook 事件**：浏览和搜索从 WaaS 2.0 发出的 Webhook 事件。每个事件都可查看其基本数据、Payload 数据和完整的投递历史记录。
- **回调消息**：查看和搜索由用户行为触发的回调消息。每条消息都可以查看其投递尝试、状态、错误信息以及消息数据，便于调试和确保正确处理事件。

## Cobo Portal Apps

该标签页用于跟踪由团队开发并部署的 Cobo Portal Apps 的 API 活动。

<Info>仅当您的团队开发了一个或多个 Cobo Portal Apps 时，该标签页才会显示。</Info>

在 Cobo Portal Apps 标签页中，您可以：

- **API 仪表板**：可视化查看每个 App 的实时 API 请求指标。
- **应用使用情况**：监控每个 App 被多少个团队和用户使用，以及启动和浏览次数，帮助评估应用的使用情况和受欢迎程度。
- **API 日志**：查看该 App 发出的所有 API 请求的详细日志。
- **Webhook 事件**：查看由该 App 触发的 Webhook 事件历史。
