Web3 钱包是一种全托管钱包的子类型，适用于您在 DeFi、NFT、dApp 等 Web3 生态中进行链上交互。您可以通过它完成智能合约调用、资产授权、消息签名等操作，而无需管理私钥或部署节点。

与侧重资产集中管理的资产钱包不同，Web3 钱包让您可以独立管理每个地址，在发起交易或链上交互时明确指定来源地址，操作更加灵活、清晰。

您可以通过 Cobo Portal 网页端或 WaaS 2.0 API 使用 Web3 钱包，也可以使用 [Cobo Connect](/cn/cobo-connect/introduction) 浏览器插件，将钱包连接到各种 dApp，安全地进行链上操作。


## 主要特点

### 完整支持链上交互  
您可以安全地进行合约调用、消息签名、授权操作，满足参与 DeFi 策略、NFT 交易、dApp 登录等多种链上使用场景。

### 私钥由 Cobo 全托管  
您无需担心私钥管理问题。Cobo 会确保私钥不可导出，并通过多重安全机制保护资金，即使发生授权误签，风险也被限制在当前地址内。

### 地址操作清晰、灵活  
您可以为每笔操作明确指定发起地址，实现资产隔离与操作可追溯，适合有合规或审计要求的场景。

### 支持多种费用策略  
您可以自由设置交易 Gas 费，也可以启用 [Fee Station](/cn/portal/fee-station/introduction)，实现 Gas 费统一代扣，有效降低使用门槛。

### 快速接入 Web3 应用  
您可以通过 Cobo Connect 等插件或 WaaS 2.0 API 将 Web3 钱包集成至各类 Web3 应用，实现安全、便捷的链上交互，而无需导出私钥。


## Web3 钱包 vs 资产钱包

Web3 钱包与资产钱包相比，主要区别如下：

| 对比维度 | Web3 钱包 |资产钱包 |
|-----------|-------------|-------------|
| 适用场景 | 适合参与 DeFi、NFT、dApp 等链上生态 | 适合集中管理资产和日常出入金 |
| 地址使用方式 | 每个地址独立操作，用户主动选择 | 出金地址抽象，系统自动分配 |
| 链上操作能力 | 支持代币转账、合约调用、签名、授权等 | 仅支持代币转账 |
| 资金归集机制 | 默认不归集，你可手动启用自动代币归集 | 默认自动归集 |
| 交易手续费 | 你可自定义 Gas 费，从转出地址扣除；也可启用 Fee Station 实现代付 | 手续费由 Cobo 统一收取和结算，用户不可控 |
| 交互方式 | Cobo Portal 网页端 / WaaS 2.0 API / 浏览器插件 | Cobo Portal 网页端 / WaaS 2.0 API |
| 代币支持范围 | 支持资产钱包的全部代币，连接 Cobo Connect 后可扩展更多 ERC-20 代币 | 支持范围固定，查看 [代币支持列表](/cn/portal/supported-tokens-and-chains) |


## Web3 钱包 vs MPC 钱包

实际使用中，Web3 钱包与 MPC 钱包在可接入功能上高度一致，在资产转账与管理方面亦具备相同能力。Web3 钱包与 MPC 钱包相比，主要区别如下：

| 对比维度 | Web3 钱包 | MPC 钱包 |
|-----------|-------------|-------------|
| 适用场景 | 无需自持密钥的 Web3 操作 | 有更高控制需求的资产托管场景 |
| 密钥控制权 | Cobo 全权托管 | 用户掌握部分密钥（分片） |
| 合规流程 | 默认接入 Cobo KYT 流程 | 可自定义，灵活性更高 |
