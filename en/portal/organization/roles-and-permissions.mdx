User roles consist of predefined sets of rules that enable you to assign specific permissions to designated members within your organization. Cobo offers preset user roles, with the option to create additional roles tailored to your specific requirements.

<Note>The following table list the current user roles and permissions.</Note>

<table class="six-columns">
  <tr>
    <th>Preset User Roles</th>
    <th>Permission Scope</th>
  </tr>
  <tr>
    <td>Viewer</td>
    <td>• Default user role after account activation.<br/>• Read-only access to asset allocation and transaction history within the organization.<br/>• Not allowed to initiate any operations.</td>
  </tr>
  <tr>
    <td>Spender</td>
    <td>• Initiate withdrawal transactions.<br/>• Interact with dApps using MPC Wallets and Smart Contract Wallets.<br/>• Same as Viewer otherwise.</td>
  </tr>
  <tr>
    <td>Staker</td>
    <td>• Stake, unstake, and withdraw assets.<br/>• Same as Viewer otherwise.</td>
  </tr>
  <tr>
    <td>Approver</td>
    <td>• Approve withdrawal transactions.<br/>• Same as Viewer otherwise.</td>
  </tr>
  <tr>
    <td>Operator</td>
    <td>• Manage wallet settings such as creating and deleting wallets as well as changing transaction policies associated with the wallet.<br/>• Same as Viewer otherwise.</td>
  </tr>
  <tr>
    <td>Admin</td>
    <td>• Manage organization settings such as freezing and unfreezing the organization, adding and removing members, creating and changing user roles, as well as changing governance policies.<br/>• Same as Viewer otherwise.</td>
  </tr>
</table>
