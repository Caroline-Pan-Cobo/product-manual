You can edit each governance policy to enforce customized approval rules. Below is a list of approval rules you can choose from for each governance policy.

| Approval Rule        | Description |
|----------------------|-------------|
| Auto Rejection       | Operations are immediately rejected without requiring approvals |
| Auto Approval        | Operations are immediately processed without requiring approvals |
| Approval Quorum      | Select a list of approvers and set up a T out of N approval quorum (e.g., 2 out of 3 approvers must approve for the operation to proceed). Approvers can be specific users or user roles. |
| Customized Workflow  | Create a customized workflow to enhance the flexibility of your action strategies. |

To edit a governance policy, click **Organization** > **Policies**. Switch to either the Members, User Roles, Organization, Wallets, or Developers tab, then click **Action** > **Edit** to open the **Edit Approval Rule** page, select and configure an **Approval Rule**, then click **Save**.
